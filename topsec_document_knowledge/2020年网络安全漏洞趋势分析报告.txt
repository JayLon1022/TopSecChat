一、前言  
随着网络和信息化的迅速发展，社交网络、移动互联网、物联网等新技术不断涌现，网络安全问题日渐突出，已成为影响经济、政治、社会等多个领域持续发展和进步的关键因素。  
网络安全漏洞主要指信息技术、产品及系统在需求、设计、实现、配置、维护和使用等过程中产生的安全缺陷。这些缺陷一旦被恶意利用，便会对信息产品或系统造成安全损害，影响正常服务运行，并危害网络安全。  
由于开发和设计等各方面的原因，软件产品存在安全漏洞在所难免。天融信阿尔法实验室特发布《2020年网络安全漏洞趋势及漏洞统计分析报告》，旨在通过对漏洞发展趋势的研究，帮助广大企事业客户和安全运维人员应对严峻的漏洞威胁。本报告重点内容分为两个部分：第一部分为2020年漏洞趋势，通过对CNVD漏洞信息库及CVE高危漏洞CVSS评分TOP100漏洞数据进行综合分析而产生。根据CNVD公开数据显示，2019年共披露漏洞16050枚，2020年共披露漏洞19930枚，同比增长24.17%。2020年高危漏洞类型分布相对集中，远程代码执行类型的漏洞占比较大，这类高危漏洞对网络空间安全的威胁远高于其他类型漏洞，说明当前网络安全态势非常严峻。  
第二部分为天融信2020年度高危漏洞预警情况概述，依据漏洞影响范围、影响对象、产生威胁等关键因素，筛选出TOP10重点漏洞。2020年度重点漏洞包括Netlogon权限提升、SMBv3远程代码执行、Weblogic IIOP远程代码执行、Exchange Server远程代码执行、Windows DNS Server远程代码执行及Apache Struts2远程代码执行漏洞等。安全漏洞数量整体呈上升趋势，其中高威胁漏洞数量和占比均有所增加，漏洞影响面逐步扩大，关键协议、服务器中间件、通用开发框架及操作系统的漏洞威胁日益严重，严重影响各类关键信息系统基础设施，基于漏洞引发的网络安全威胁应引起高度警惕。

二、2020年漏洞趋势  
1. CNVD漏洞库2020年漏洞统计概况  
漏洞的统计与评判是评估网络安全情况的重要指标。天融信阿尔法实验室参考CNVD漏洞数据库的数据，对2020年披露的漏洞进行了全方位的统计分析，具体统计情况如下：2019年共披露漏洞16050枚，2020年共披露漏洞19930枚，同比增长24.17%。其中，高危漏洞6903枚，同比增长42.36%；中危漏洞10616枚，同比增长10.91%；低危漏洞2411枚，同比增长48%。各级别漏洞数量均处于近10年的新高。  

近十年漏洞数量走势图

1) 漏洞威胁等级统计  
根据2020年1-12月漏洞引发的威胁严重程度统计，低危漏洞占10.6%，中危漏洞占52.6%，高危漏洞占36.8%。  

2) 漏洞利用攻击位置统计  
根据2020年1-12月漏洞引发威胁的统计，远程攻击占比约80.2%，本地攻击约占17.9%，其他攻击为1.8%。由此可见，远程攻击是主要的漏洞攻击手段，也是我们主要防范的漏洞攻击方式。  

3) 漏洞影响对象类型统计  
根据2020年1-12月漏洞引发威胁的统计，受影响的对象大致可分为九类：操作系统漏洞、应用程序漏洞、WEB应用漏洞、数据库漏洞、网络设备漏洞、安全产品漏洞、智能设备漏洞、区块链公链漏洞、区块链联盟链漏洞。其中，应用程序漏洞占48%，WEB应用漏洞占27.7%，操作系统漏洞占10.3%，网络设备漏洞占6.8%，智能设备漏洞占2.1%，区块链公链漏洞占1.9%，安全产品漏洞占2%，数据库漏洞占1.3%。  

4) 漏洞产生原因统计  
根据2020年1-12月漏洞产生原因的统计，设计错误导致的漏洞占比65.4%屈居首位，输入验证错误导致的漏洞占比25.4%位居第二，边界条件错误导致的漏洞占比5.3%位居第三。后面的意外情况处理错误、访问验证错误、配置错误、竞争条件、环境错误等分别占0.1%、1.4%、0%、0.2%、0%、2.1%。  

5) 漏洞引发威胁统计  
根据2020年1-12月漏洞引发威胁的统计，未授权信息泄露占比28.3%屈居首位，管理员访问权限获取漏洞占比24.6%位居第二，拒绝服务占比16.2%位居第三。其他如普通用户访问权限获取、未授权信息修改及未知等分别占5%、12.5%、11.7%、1.7%。  

2. 2020年CVE TOP100漏洞统计概况  
通过对CVE在2020年公布的漏洞按CVSS评分高低进行排序，我们筛选了CVSS基本评分最高的前100个漏洞进行统计分析。此次统计分析主要从漏洞所影响厂商、影响平台、攻击途径、披露时间、漏洞类型以及POC公开情况等六个方面展开。结果显示，漏洞影响厂商前三名分别是三星、高通及Oracle。从影响平台进行统计，受影响的平台大致可分为五类：PC端平台、移动端平台、硬件设备平台、跨平台及其他平台。其中，PC端平台漏洞占44%，占据首位。由此可见，漏洞依然集中在传统厂商的设备和产品中，且主流系统和产品面临的漏洞威胁和安全风险较大。  

高危漏洞的披露时间看，7月份共披露高危漏洞17个，位居全年第一。在TOP100漏洞中，约16%的高危漏洞存在公开POC，这一数据占比不高，但公开POC为攻击者提供了便利条件，一旦被攻击者率先掌握漏洞利用方式并实现攻击工具，将对相关软硬件设备造成重大安全危害，形成威胁。为避免类似事件，需由软硬件厂商及安全厂商携手建立良好的安全生态。  

从攻击途径看，可被远程利用的漏洞占比约79%，本地利用的漏洞约占21%。来自互联网的漏洞依旧是主要的攻击手段。在TOP100高危漏洞中，远程代码执行类型的漏洞占比71%。漏洞类型分布相对集中，表现为远程代码执行类漏洞占比较大，这类高危漏洞对网络空间安全的威胁远高于其他类型漏洞，说明当前严峻的网络安全态势。  

具体统计分析结果如下：  
1) 漏洞影响厂商分布情况  
根据2020年1-12月CVE披露漏洞危害程度前100例所影响的厂商情况进行统计，前三名分别是三星、QUALCOMM及Oracle。其中，三星厂商的产品占比达到53%，QUALCOMM的产品占17%，Oracle的产品占7%。  

2) 高危漏洞披露时间趋势图  
根据2020年1-12月CVE披露漏洞危害程度前100例披露时间进行统计，在2020年全年中，7月份披露17个TOP100漏洞占比17%位居第一，4月和3月份分别披露13、12个漏洞位居第二与第三，而2月份未有TOP100漏洞被披露。  

3) 攻击途径概况  
根据2020年1-12月CVE披露漏洞危害程度前100例攻击途径进行统计，其中来自远程攻击占比约79%，本地攻击约占21%。由此可见，来自公网的攻击是主要的漏洞攻击手段，远程攻击也是我们主要防范的漏洞攻击手段。  

4) 漏洞影响平台分类  
根据2020年1-12月CVE披露漏洞危害程度前100例所影响的平台进行统计，受影响的平台大致可分为五类：PC端平台、移动端平台、硬件设备平台、跨平台及其他平台。其中，PC端平台漏洞占44%，移动端平台漏洞占32%，硬件设备平台漏洞占9%，跨平台漏洞占7%，其他漏洞占8%。  

5) 漏洞类型统计概况  
根据2020年1-12月CVE披露漏洞危害程度前100例类型进行统计，其中远程代码执行漏洞占比最多，以71%位居首位，而权限提升、内存破坏、命令执行以及其他漏洞分别占比10%、7%、5%、4%、3%。  

6) TOP100 POC公开情况统计  
根据2020年1-12月CVE披露漏洞危害程度前100例POC公开情况进行统计，其中未公开POC占比最大，达到45%，公开POC的仅有16%，而39%的漏洞未明确POC公开情况。  

三、漏洞预警统计概况  
天融信阿尔法实验室在2020年共发布高危漏洞风险提示通告61条，涉及多家厂商的软件产品，由漏洞引发的安全威胁多种多样。统计结果显示，主流操作系统是漏洞高发产品。2020年针对Microsoft厂商的漏洞预警次数达14次，其中Windows系统的漏洞占大多数。Weblogic、WebSphere、SMB协议、OpenSSL等关键基础设施漏洞也是受关注度较高的方向。  

2020年预警的漏洞中，代码执行类漏洞占比最高，达到58%。这一类漏洞也是APT攻击者的重要目标和攻击武器，攻击者利用这类漏洞可以远程执行任意代码或指令，有些漏洞甚至无需用户交互即可实现远程代码执行，对目标网络和信息系统造成严重影响。具体预警统计分析情况如下：  

1. 漏洞厂商情况  
在2020年内发布的61条漏洞通告中，针对Microsoft厂商的漏洞预警次数最多，为14次，占比约23%。针对Apache厂商的漏洞预警为8次，占比约13%位居第二，而针对Oracle和VMware厂商的各5次，并列位居第三。  

2. 漏洞威胁情况  
在2020年发布的61条漏洞通告中，所通告的漏洞可分为8大类，分别是代码执行漏洞、拒绝服务漏洞、权限绕过漏洞、文件操作类漏洞、命令执行漏洞、虚拟机逃逸漏洞、注入漏洞及其他漏洞，其中代码执行漏洞占58%位于首位，拒绝服务、权限绕过及其他漏洞分别占比8%共同位于第二位，虚拟机逃逸与注入类漏洞分别占比3%位于末位。

四、漏洞预警TOP10漏洞回顾  
1. Microsoft NetLogon远程权限提升漏洞  
1) 漏洞描述  
该漏洞是Windows Server在实现登录验证的AES-CFB8加密算法初始化IV时不当使用随机数导致。在初始化IV时有1/256的概率使得IV为全0，导致加密得到的密文全0，最终导致身份验证被绕过。未经身份验证的远程攻击者利用此漏洞，无需身份验证即可获取域控制器的管理员权限。该漏洞影响版本广泛，Windows Server 2008、2012、2016、2019等多个Windows Server版本受到此漏洞影响。攻击者利用此漏洞，无需身份验证即可获取域控制器的管理员权限并在设备上运行经过特殊设计的应用程序。成功利用该漏洞可能会对受害者资产造成极大的破坏。  

2. Microsoft SMBv3协议远程代码执行漏洞  
1) 漏洞描述  
本次漏洞存在于微软SMBv3.0协议中，由SMBv3处理恶意压缩数据包时进入错误流程造成。未经身份验证的攻击者可以通过利用该漏洞，向存在漏洞的受害主机的SMBv3服务发送特殊构造的数据包，从而远程执行任意代码。该漏洞主要影响Windows 10及Windows Server下的多个版本，影响面广泛，推测可能在未来会成为恶意软件和攻击者广泛利用的漏洞。  

3. Apache Struts2 S2-061远程代码执行漏洞  
1) 漏洞描述  
Apache Struts 2最初被称为WebWork 2，它是一个简洁的、可扩展的框架，用于创建企业级Java web应用程序。该框架旨在简化整个开发周期。2020年12月08日披露的S2-061 Struts远程代码执行漏洞（CVE-2020-17530），在使用某些标签等情况下可能存在OGNL表达式注入漏洞，从而造成远程代码执行，风险极大。该漏洞主要影响Apache Struts 2.0.0-2.5.25，而Apache Struts2作为世界上最流行的Java Web服务器框架之一被各大公司采用，因此该漏洞影响面极其广泛。  

2) 数据分析  
天融信安全云服务运营中心通过风险探知系统对我国境内部署Apache Struts2的服务器进行统计，结果显示我国境内的Apache Struts2服务器约有80672台。按区域统计，排名前三的省份是北京市17339台，浙江省32506台，广东省14266台。  

4. Weblogic IIOP远程代码执行漏洞  
1) 漏洞描述  
CVE-2020-2551存在于Weblogic WLS组件IIOP协议。互联网内部对象请求代理协议（IIOP）是一个实现互操作性的协议，使得由不同语言编写的分布式程序可以在因特网上实现交流。它是行业战略性标准，公用对象请求代理程序结构（CORBA）中至关重要的部分。该漏洞影响Oracle WebLogic Server 10-12版本，攻击者可以通过IIOP协议远程访问Weblogic Server上的远程接口，传入恶意数据，从而获取服务器权限并在未授权情况下远程执行任意代码。由于IIOP协议以Java接口的形式对远程对象进行访问且默认启用，Weblogic作为Oracle的主要产品之一被广泛使用，该漏洞在全球范围内造成了极大的影响。  

2) 数据分析  
天融信安全云服务运营中心通过风险探知系统对世界范围内部署Weblogic的服务器进行统计，结果显示Weblogic服务器约有7987台。按统计，排名前五的国家是中国、法国、美国、越南、印度。  

5. Weblogic Console HTTP协议代码执行漏洞  
1) 漏洞描述  
WebLogic是美国Oracle公司出品的一个应用服务器，确切地说是一个基于Java EE架构的中间件。WebLogic用于开发、集成、部署和管理大型分布式Web应用、网络应用和数据库应用。WebLogic远程代码执行漏洞CVE-2020-14882和CVE-2020-14883两个高危漏洞POC已公开，未经身份验证的攻击者可以通过构造恶意HTTP请求利用该漏洞，成功利用此漏洞可能接管Oracle WebLogic Server。Weblogic作为Oracle的主要产品之一，是商业市场上主要的Java（J2EE）应用服务器软件之一，全球范围内被大量使用，该漏洞影响了Oracle Weblogic Server多个版本，造成了极大的影响。  

6. Microsoft Exchange Server远程代码执行漏洞  
1) 漏洞描述  
此漏洞源于Microsoft Exchange控制面板（ECP）组件中使用静态密钥造成的。根据POC，本次利用需要攻击者拥有一个邮箱账号并使用该账号登录服务器，在登录过程中获取凭证，进而通过获取到的凭证构造出恶意数据，最终向存在漏洞的ECP组件发送包含恶意数据的请求进行攻击。执行此攻击无需用户交互，并且会以系统级别权限执行，从而完全控制Exchange服务器。该漏洞影响Microsoft Exchange Server多个版本。  

2) 数据分析  
天融信安全云服务运营中心通过风险探知系统对我国境内部署的Microsoft Exchange服务器进行抽样统计，结果显示我国境内暴露在互联网中的Microsoft Exchange服务器有1.2万余台，这些服务器开放最多的端口是25和110，其次是587、143和995。按区域统计，排名前三的省份是广东省2948台，上海市2395台，香港特别行政区2109台。  

7. Windows DNS Server远程代码执行漏洞  
1) 漏洞描述  
此漏洞源于Microsoft DNS Server实现中的缺陷。利用此漏洞，未经身份验证的攻击者通过向Microsoft DNS Server发送恶意请求，可执行任意代码。微软认为这是一个可蠕虫攻击的漏洞，可以在易受攻击的计算机之间传播，而不需要用户交互。该漏洞影响程度较为严重，建议所有运行DNS服务器的用户尽快更新安全补丁。此外，该漏洞仅影响Microsoft的Windows DNS Server实现，不会影响Windows DNS客户端。  

2) 数据分析  
天融信安全云服务运营中心通过风险探知系统对世界范围内部署Microsoft DNS的服务器进行统计，结果显示Microsoft DNS服务器约有230338台。按统计，排名前五的国家是美国、中国、土耳其、英国、德国。  

8. SQL Server远程代码执行漏洞  
1) 漏洞描述  
2020年2月16日，在微软2月份的安全更新中发现SQL Server远程命令执行漏洞（CVE-2020-0618）。SQL Server是Microsoft公司推出的关系型数据库管理系统，是一个全面的数据库平台。要利用该漏洞需要经过身份验证后，向受影响的SQL Server的报告服务发送精心编制的页面请求，成功利用此漏洞的攻击者可以在SQL Server服务的上下文中执行代码。该漏洞影响Microsoft SQL Server多个版本，具有较大的威胁。  

2) 数据分析  
天融信安全云服务运营中心通过风险探知系统对我国境内部署SQL Server数据库的服务器进行统计，结果显示我国境内的SQL Server服务器有8.2万余台。按区域统计，排名前三的省份是浙江省3万余台，北京市1万6千余台，广东省1万3千余台。  

9. F5 BIG-IP TMUI远程代码执行漏洞  
1) 漏洞描述  
F5 BIG-IP是美国F5公司的一款集成网络流量管理、应用程序安全管理、负载均衡等功能的应用交付平台。攻击者利用该漏洞，通过向TMUI发送恶意攻击请求，从而执行任意系统命令、创建或删除文件、禁用服务，以及执行任意Java代码，最终完全获取系统权限。该漏洞影响F5 BIG-IP众多版本，在受影响的版本中，F5 BIG-IP Appliance模式下的BIG-IP系统也受到漏洞影响，但不影响数据面，只影响控制面板。  

2) 数据分析  
天融信安全云服务运营中心对中国境内的F5 BIG-IP设备进行了抽样数据统计及分析，数量约为1.4万余台。排名前五的省份或地区分别为北京市、广东省、上海市、香港特别行政区、台湾省。  

10. Apache Dubbo远程代码执行漏洞  
1) 漏洞描述  
Apache Dubbo是一个高性能的服务框架，使得应用可以通过高性能的RPC实现服务的输出和输入功能，可以和Spring框架无缝集成。Dubbo Provider存在反序列化漏洞，攻击者可以通过RPC请求发送无法识别的服务名称或方法名称以及一些恶意有效载荷，当恶意参数被反序列化时，可以造成远程代码执行。Apache Dubbo作为一个优秀的服务框架被广泛使用，该漏洞影响Dubbo多个版本，在全球范围内具有较大的威胁。  

2) 数据分析  
天融信安全云服务运营中心通过风险探知系统对我国境内部署Apache Dubbo的服务器进行统计，结果显示我国境内的Apache Dubbo服务器约有400台。按区域统计，排名前五的省份或地区为广东省、浙江省、北京市、上海市、陕西省。  

五、总结  
与往年相比，2020年漏洞数量快速增长，漏洞的激增表明企业在大规模开展业务的同时难以管理业务自身带来的网络风险。近年来，由漏洞引发的各种勒索病毒事件、APT攻击事件、数据泄露事件层出不穷，对企业以及个人安全造成了严重威胁。  
纵观2020年的高危漏洞，具有以下趋势：第一是漏洞跨度广泛，主流操作系统、流行网络协议、底层TCP/IP协议库、知名WEB容器和开发框架以及网络安全设备均被发现存在安全漏洞。第二是漏洞类型广泛，远程代码执行、反序列化、命令执行及权限提升等严重影响的安全漏洞占比较高。第三，部分漏洞被恶意利用的可能性更大，据统计，相当一部分高危漏洞的POC在互联网上公开流传，攻击者基于这一类信息可以进一步开发漏洞利用工具，从而给网络安全造成实际威胁。  
基于2020年安全漏洞的发展趋势，我们建议：安全运维人员在部署网络服务时应做好权限访问控制，减少对外部网络暴露组件接口的情况；关注网络安全资讯，及时排查受漏洞影响的网络服务；终端用户应及时进行系统及应用软件的安全更新，避免遭受公开漏洞的安全威胁；安全厂商除了关注安全产品的核心功能外，还应特别重视产品自身的安全问题，避免因产品漏洞导致安全堡垒被攻破。可以预见，在即将到来的2021年中，漏洞数量，尤其是高危漏洞的数量将继续呈现增长态势。随着安全人员对其关注度的日益增加，相关漏洞也将不断被披露。面对激增的漏洞数量，如果企业难以解决这些漏洞带来的问题，则企业安全将受到严重损害。为了从根源上避免安全事件的发生，应减少漏洞的产生。这需要开发人员在掌握编程能力的同时，具备安全开发意识。同时，应将代码审计等环节纳入软件的开发过程中。然而，漏洞并不会被彻底消灭，所谓的安全性不是指“安全”或“不安全”，而是取决于对安全事件的响应速度。只有提高漏洞管理效率，才是最有效的安全处理法则。作为国内领先的网络安全、大数据与云服务提供商，天融信始终以捍卫国家网络空间安全为己任，创新超越，持续为客户构建更加完善的网络安全防御能力，为数字经济的发展保驾护航。天融信将充分发挥自身优势，在保障客户网络安全的同时，努力践行领军企业的社会责任与担当，为国家网络安全整体能力建设贡献力量，为实施网络强国战略做出企业贡献。
